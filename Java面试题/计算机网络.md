## 网络互联模型

为了更好的促进互联网络的研究和发展，国际标准化组织在 1985 年制定了网络互联模型 - OSI 七层参考模型（Open-System Interconnect Reference Model）。
![[Pasted image 20251208183715.png]]

但是由于ISO组织太理论化，这个标准并没有应用到实践中，真正使用的是TCP/IP四层模型。

## 集线器

同轴电缆也会把数据进行传递给所有连接的设备，只要不到电阻就会进行传递，到末端之后会返回。

同轴电缆升级版。

会将收到的数据转发给所有连接的设备（同网段下！）

进行通信时会先进行ARP广播（ARP寻址），获取到目标IP的MAC 地址（网卡地址），这个ARP广播在经过集线器时也会转发给所有连接的设备。即使获取到了目标的MAC地址，集线器也会把数据发给所有连接的设备。如果IP地址不是被寻找的IP地址 ，那么会丢掉收到的数据包。

半双工通信
不安全
容易冲突
不安全

## 网桥

如果集线器链接了1000台设备，如果进行TCP建立连接时，会进行ARP寻址，将这个包发送给1000台设备，就算已经知道了设备的MAC地址也会把数据包发送给所有的设备。效率低。

网桥可以通过自学习获取每个接口那测的设备的MAC地址，从而起到隔绝冲突域的作用。内部会维护一个MAC地址表 （接口 - MAC地址）。会阻拦部分流量。

弊端：
接口不够；
隔绝作用有限（同侧还是会有冲突域）。
